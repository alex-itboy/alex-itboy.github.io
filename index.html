File sharing using ownCloud deployed with Docker

Why I need it

We all have the time that needs to share files with friends and teammates. In this semester we have some little projects that require teamwork, so we have to find a way to share files.

Actually, almost all of our files that need to be shared are source code files, so git is my first thought. But git is not that easy to use, and I don't really care about version control. So no git then.

My second thought is ownCloud. I use ownCloud for syncing files between two laptops before and ownCloud left me a very good impression. For those who don't know about ownCloud, here is the description from the official website:

ownCloud is a self-hosted file sync and share server. It provides access to your data through a web interface, sync clients or WebDAV while providing a platform to view, sync and share across devices easily — all under your control. ownCloud’s open architecture is extensible via a simple but powerful API for applications and plugins and it works with any storage.

Yeah, ownCloud is quite awesome, it has many features that you can discover on their website.

And luckily, one of my teammates has a VPS, not a powerful one but enough for ownCloud because ownCloud don't require much, I run my own ownCloud on my Raspberry Pi 3 Model B perfectly ( for 2 laptops and my phones and tablets).

Deploying ownCloud on Centos using Docker

ownCloud requires PHP and other stuff. It can run on Linux and Windows(not since 8.1), and there are so many ways to install it.

On their website, they provide several ways to install ownCloud including Archive File ,Web Installer, Packages and Appliances.

Archive File and Web Installer require a pre-installed PHP environment, which is not what I want. What I want is a simple way to deploy it and make it runs. Packages are for popular Linux distributions and through packages we can easily update ownCloud to the latest version. Appliances are configured ownCloud in virtual machine images, the EASIEST way to deploy ownCloud but not for VPS.

Actually, packages sound great for me, but I use Docker to deploy ownCloud on my Raspberry Pi and Docker is such a handy tool so I decided to use Docker to deploy ownCloud on the VPS.

Because the package name docker has already token, so you have to install Docker by the name docker-io

yum install docker-io

Start docker service

service docker start

Pull owncloud image

docker pull owncloud

In order not to interfere whatever is running on the VPS, publish the docker port 80 to 3080 and 443 to 3081 and to save data or persistent data, mount /owncoud_data to /var/www/html

docker run -d --name=owncloud -p 3080:80 -p 3081:443 -v /owncloud_data:/var/www/html/data owncloud

And your ownCloud server should be running, you can see it on http://your_vps_ip:3080/. Open it and you will see a setup page, set up an admin user and choose SQLite as the database, then you are all set.

To autostart ownCloud after reboot, run this command after booting (add it to a script that will autorun)

docker start owncloud

Ways to use ownCloud

One account for the team

This is how we use it, which is the whole team use the same account. I may have mentioned above that I choose ownCloud for its simplicity, and this way makes it much easier to use.

  • Install the ownCloud desktop Client for syncing files automatically
  • Signing the same account in the client and sync all files
  • Create a folder for each team members

When we want to share a file, drop it into your folder(your ownCloud client will monitor the folder and upload the new file to the server). Tell others you have uploaded the files(their ownCloud client will download the file from the server in the background), and they can see your file in their syncing folder in a minute.

By doing this, we don't have to use the web browser to share files with other teammates, but there will be no access control and others may mess up your files.

One account each team member

This is the proper way to use it, but a little more inconvenient.

To share files with others, you first upload your file to your folder(by dropping it to the syncing folder or upload through the web) then use web client to share with other users. All the files that shared with you will be added to your folder then download by ownCloud client.

However, this is too much trouble, we don't want to use a web client to share our files. But some access control will be very nice.

Apart from that...

ownCloud is a very nice and powerful platform, files sharing is one of its many features, you guys should really check it out!
